1. Problem & Opportunity

Problem
大学生 / 考研党积累了大量TXT/PDF 格式的专业课笔记、真题、复习资料，文件分散且单个体积较大，手动翻找效率极低。
系统自带搜索工具（如 Windows 搜索、Mac Spotlight）检索速度慢，大文件场景下响应延迟明显，且无法精准定位关键词所在的文本片段。
在线文本检索工具存在数据泄露风险，不适合存储敏感的考研复习资料；现有本地检索工具功能简陋，缺乏高效索引和算法优化，难以满足高频检索需求。
手动翻找文件会浪费大量复习时间，严重影响备考进度。

Opportunity
利用C/C++ 底层高性能特性（如内存映射、倒排索引、KMP 字符串匹配算法），实现毫秒级本地文本检索，解决大文件检索慢的核心痛点。
结合 Python 快速开发优势，构建友好的交互界面，降低用户使用门槛，同时体现跨语言协作的工程能力，适合作为考研复试展示项目。
产品可直接服务于考研备考场景，帮助用户高效检索知识点，同时作为个人项目在复试中展示底层编程与算法优化能力，具备“实用 + 复试加分” 的双重价值。

2. Target Audience & User Persona

Target Audience
核心用户：备考研究生的大学生（尤其是计算机 / 理工科专业），拥有大量专业课笔记、真题资料，需要高频检索知识点。
次要用户：需要管理大量文本资料的大学生（如论文文献整理、课程笔记检索），追求本地、安全、高效的检索体验。

User Persona
姓名：李明
身份：XX学校XX专业大三学生，正在备考研究生
痛点：积累了 500MB + 的专业课笔记（TXT/PDF 格式），每次复习需要手动翻找知识点，系统搜索耗时久，关键词定位模糊，影响复习效率。
需求：希望有一款本地、安全、高效的文本检索工具，能快速定位笔记中的关键词，支持大文件检索，操作简单，不依赖网络。

3. Metrics and Goals

业务目标
核心目标：将用户文本检索时间从分钟级缩短至秒级，大幅提升考研复习效率。
辅助目标：项目可作为考研复试展示作品，体现底层编程与算法优化能力，成为简历 / 复试中的亮点。

技术指标
检索响应时间：针对 100MB 文本文件，关键词检索响应时间≤1 秒。
索引构建时间：100MB 文本文件的倒排索引构建时间≤30 秒。
检索准确率：关键词匹配准确率≥95%（排除无意义片段）。
支持文件格式：TXT、PDF（基础文本内容提取）。
数据安全：所有数据本地存储，不上传云端，确保用户资料安全。

4. Use Cases / Features

Use Cases
文件导入：用户选择本地 TXT/PDF 文件，系统自动解析文本内容并构建倒排索引。
关键词检索：用户输入单个关键词，系统快速检索并展示包含该关键词的所有文本片段，高亮显示关键词。
高级检索：用户使用“与/或/非”逻辑组合关键词（如 “数据结构 AND 二叉树”），精准定位目标内容。
结果查看：用户点击检索结果片段，跳转至对应文件位置，查看完整上下文。
索引管理：用户可手动删除、重建索引，优化检索性能，释放存储空间。

Features
核心功能
高性能检索引擎：基于倒排索引和 KMP 算法，实现毫秒级关键词匹配。
大文件处理：通过内存映射（mmap）技术，高效读取 100MB + 文本文件，减少 IO 开销。
本地数据存储：所有索引和数据存储在本地，确保用户资料安全。

辅助功能
文件解析：支持 TXT/PDF 文件的文本内容自动提取，无需用户手动处理格式。
结果高亮：检索结果中高亮显示关键词，便于用户快速定位目标内容。
简易 GUI 界面：Python 实现图形化交互界面，操作直观，无需命令行操作。
索引管理：支持索引的创建、删除、重建，适配文件更新场景。

5. Epics & User Stories

Epic 1：核心检索引擎构建（C/C++ 底层模块）
User Story 1：作为开发者，我需要实现倒排索引数据结构，以便高效存储和检索关键词与文本片段的映射关系。
User Story 2：作为开发者，我需要实现KMP 字符串匹配算法，以便在文本中快速定位关键词，提升检索速度。
User Story 3：作为开发者，我需要实现内存映射（mmap）文件读取，以便高效处理大体积文本文件，减少磁盘 IO 开销。
User Story 4：作为开发者，我需要将核心检索逻辑封装为动态库，以便 Python 上层模块调用，实现跨语言协作。

Epic 2：文件导入与解析（Python 上层模块）
User Story 1：作为用户，我希望能导入本地 TXT 文件，以便系统读取内容并构建索引。
User Story 2：作为用户，我希望能导入本地 PDF 文件，以便系统自动提取文本内容并构建索引。
User Story 3：作为用户，我希望系统能自动解析文件格式，无需手动处理编码或排版问题。

Epic 3：用户交互界面（Python 上层模块）
User Story 1：作为用户，我希望有一个图形化界面，以便输入关键词并查看检索结果。
User Story 2：作为用户，我希望检索结果中高亮显示关键词，以便快速定位目标内容。
User Story 3：作为用户，我希望点击检索结果能跳转至对应文件位置，查看完整上下文。

Epic 4：索引管理与优化
User Story 1：作为用户，我希望能手动重建索引，以便在文件更新后更新检索数据。
User Story 2：作为用户，我希望能删除无用索引，以便释放本地存储空间。
User Story 3：作为开发者，我需要优化索引存储结构，以便减少内存占用，进一步提升检索速度。
